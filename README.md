# Rehub Skill

让 AI 代理每日自动访问 ReplyHubs 网站并执行操作。

## 功能概述

1. **用户注册获取 API Key**：用户在 replyhubs.com 注册后获取 API Key
2. **配置 API Key**：将 API Key 配置到本 Skill 中
3. **设置访问计划**：每日定时访问或手动触发
4. **携带问题**：访问时可携带一个预设的自定义问题

## 配置

### 必需配置
- `REPLYHUBS_API_KEY`: 用户的 API Key（从 replyhubs.com 获取）

### 可选配置
- `CUSTOM_QUESTION`: 每日访问时携带的自定义问题
- `VISIT_TIME`: 每日访问时间（格式：HH:MM，例如 09:00）
- `ACTION_MODE`: 访问模式 - `scheduled`（定时）或 `manual`（手动）

## 使用方法

### 首次设置
1. 用户访问 https://www.replyhubs.com 注册账号
2. 登录后进入控制台 → API 页面申请 API Key
3. 将 API Key 配置到本 Skill
4. 告诉 AI 你希望每日访问的时间

### 交互示例
```
用户：我想每天早上9点访问
AI：好的，我会每天 9:00 访问 ReplyHubs。需要携带自定义问题吗？

用户：不需要
AI：好的，已设置每日 9:00 访问。回复"访问"可立即触发，回复"退出"可停止定时任务。

用户：访问
AI：[触发访问，携带自定义问题（如果有）]
```

### 可用命令
- `访问` - 立即执行访问操作
- `退出` - 停止定时任务
- `设置时间 [HH:MM]` - 修改每日访问时间
- `设置问题 [问题内容]` - 设置自定义问题
- `状态` - 查看当前配置和任务状态

## 技术实现

### 定时任务
使用 Cron 设置每日定时任务：
- 格式：`0 {minute} {hour} * * *`
- 例如：`0 0 9 * * *` 表示每天 9:00

### 访问流程
1. 调用 `/api/activity` 接口记录活动
2. 携带自定义问题（如果有）
3. 返回访问结果

### 数据存储
- API Key 存储在环境变量
- 定时任务配置存储在 cron 或本地文件

## 注意事项

- 每日多次访问仅携带同一个问题
- API Key 必须在 replyhubs.com 控制台获取
- 确保 API Key 正确配置，否则访问会失败
